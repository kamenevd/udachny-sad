## 🔧 Топ-5 улучшений для Production (консенсус)

1. **Exponential Backoff в F.1** — ретраи очереди идут подряд, нет задержки. Добавить `retries * 1000 * 2^n` и `try/finally` для сброса `flushing` флага.

2. **`prefers-contrast: more` в G.1** — авто-детект системного High Contrast. Убрать `!important`, повысить специфичность через `:root[data-hc="true"]`.

3. **`continuous: true` в G.2** — на iOS распознавание обрывается через ~1 сек. Добавить таймаут молчания и очистку interim results.

4. **Расширить схему I.1** — добавить `spacingCm`, `daysToHarvest`, `sunRequirements`, `waterNeeds`, `companionPlants`. Парсимые даты цветения (`bloomStart: 6`).

5. **Conflict resolution в F.1** — сейчас implicit "last write wins". Добавить версионирование/ETag для multi-device редактирования.

## Детальные замечания по задачам с кодом

### F.1 — Offline Sync Engine & Queue
**⚠️ Выполнено, но есть замечания**

1. **Нет exponential backoff** — ретраи идут подряд, без задержки. При сетевых проблемах это создаёт лишнюю нагрузку.
2. **Poisoned messages просто удаляются** после `MAX_RETRIES=5`. Нет dead-letter queue для ручного разбора пользователем.
3. **Conflict resolution = last write wins** (implicit). Нет версионирования/ETag — риск потери данных при одновременном редактировании.
4. **`queue.ts` показан только head** — не видно реализации `put/remove/count/subscribeQueue`, атомарности транзакций, обработки `IDBTransaction` ошибок.
5. **Нет батчинга** — каждая мутация отдельный запрос к PB. Можно собирать в батчи (PB поддерживает batch).

### G.1 — High Contrast Mode
**⚠️ Выполнено, но есть замечания**

1. **`!important` везде** — делает CSS хрупким, сложно переопределять компонентам. Лучше: повысить специфичность селекторов (`:root[data-hc="true"] .btn`).
2. **Нет `prefers-contrast: more`** — пользователь с системным HC не получит тему автоматически.
3. **`font-size: 19px` на `:root`** — ломает `rem`-верстку глобально. Лучше масштабировать через `--scale: 1.18` и умножать в компонентах.
4. **Контраст не верифицирован** — заявлено ≥7:1, но нет автотеста (axe / color-contrast checker в CI).
5. **`box-shadow: none !important` на `*`** — убирает фокус-ринг у нативных элементов, но ниже задаётся свой `outline`. Порядок важно.

### G.2 — useSpeechRecognition
**⚠️ Выполнено, но есть замечания**

1. **`continuous: false`** — речь обрывается на паузе >~1 сек. Для дачников (шум, поиск слов) нужен `true` + ручной `stop`.
2. **Interim results накапливаются в `transcript`** — при долгом говорении строка растёт, нет очистки старых interim.
3. **Нет `onnomatch` / `onaudiostart`** — UX: пользователь не видит, что микрофон слушает.
4. **Ошибка `not-supported` — просто строка**. Лучше вернуть `{ supported: false }` сразу в стейте, не ждать `start()`.
5. **Нет очистки `transcript` при новом `start()`** — если пользователь нажал «Говорить» дважды, старый текст останется.

### I.1 — Plant Varieties (72 сорта)
**⚠️ Выполнено, но есть замечания**

1. **Схема бедная**: нет `spacingCm`, `daysToHarvest`, `companionPlants`, `avoidPlants`, `sunRequirements`, `waterNeeds`.
2. **`bloom` — строка** («июнь–июль»). Непарсибельно для календаря. Лучше: `bloomStart: 6, bloomEnd: 7` (месяцы) или ISO-даты.
3. **Нет уникальных ID** — сложно ссылаться из посадок, нет стабильного ключа для кэша.
4. **Только annuals в head** — где perennials (ягоды, деревья)? 72 сорта — мало для полного каталога.

### I.2 — SmartBrush
**⚠️ Выполнено, но есть замечания**

1. **Алгоритм min-area bounding rectangle (rotated rect) не показан** — это ядро задачи. В head только `simplifyStroke` и `bboxDiagonal`.
2. **Нет валидации минимума точек** — для прямоугольника нужно ≥3 неколлинеарные точки. При 2 точках — дегенерация.
3. **`minGapM = 0.05` (5 см)** — может быть крупно для маленьких грядок (1×1 м). Сделать адаптивным: `max(0.02, diagonal * 0.02)`.
4. **`closed` определяется как «конец рядом с началом»**, но нет порога в коде (не видно).
5. **Нет обработки самопересечений** — дачник может нарисовать «восьмёрку».

### J.1 — Notification Scheduler
**⚠️ Выполнено, но есть замечания**

1. **Только localStorage** — не работает в background (Service Worker нужен для `showNotification` при закрытой вкладке).
2. **Нет `Notification.permission` handling** — код предполагает, что разрешение уже дано.
3. **`repeatEveryDays` пропускает просроченные** — хорошо, но нет `maxCatchUp` (если офлайн месяц, не спамить все 30).
4. **Head только 50 строк** — не видно `getDueReminders`, `markFired`, таймеров/alarms.
5. **Нет типизации `eventDate` в reminder** — как связать с календарём?

### J.2 — useWateringReminder
**⚠️ Выполнено, но есть замечания**

1. **Порог 7 дней hardcoded** — у капусты и кактусов разные нужды. Нужен `wateringIntervalDays` на сорт/посадку.
2. **Не интегрирован с J.1** — не создаёт напоминание автоматически, только показывает флаг.
3. **`eventType === 'watering'` — string literal** — опечатка в типе события сломает логику. Нужен `enum` / константа.
4. **Нет таймзоны** — `Date.now()` vs `eventDate` (строка). Если `eventDate` без TZ — баг на переходе летнее/зимнее.
5. **`events` может быть большим** — фильтрация на каждом рендере. Лучше мемоизировать `waterings` отдельно.

### K.2 — useRealtimeJournal
**⚠️ Выполнено, но есть замечания**

1. **`subscribe('*')` — подписка на все коллекции** — oversubscription. PB позволяет фильтр на сервере: `subscribe('*', {
